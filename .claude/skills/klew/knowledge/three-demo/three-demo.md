# three-demo — app knowledge

three.js (3D WebGL); meshes addressed by name, world position projected through the camera to screen sample for the klew **scene tier** (see `references/selector-policy.md`). The
objects have no DOM element, so they are cached as `tier: scene` and clicked via
the app's own scene model — never a hardcoded pixel. Sample: `e2e/scene/three.html`.

## Notes & gotchas

- Requires the scene instance (`window.__three` = { scene, camera, renderer }) to be exposed.
- Every scene entry is `a11y_flag: true` — canvas objects are absent from the
  accessibility tree.
- Act via `scripts/scene_click.py` or the generated `three-demo.pom.ts`.

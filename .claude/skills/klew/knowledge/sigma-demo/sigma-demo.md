# sigma-demo — app knowledge

A Sigma.js (WebGL) graph, used to exercise klew's **scene tier**. The graph nodes
are drawn to a `<canvas>` and have no DOM element, so they are cached as
`tier: scene` entries addressed by their **label** and clicked via the app's own
`graphToViewport` transform (see `references/selector-policy.md` → Scene tier).

The app exposes its Sigma instance on `window.__sigma`; the sample lives in
`e2e/sigma/app`. A plain HTML search box (`data-test="node-search"`) is the
DOM-control fallback and is cached with an ordinary tier-3 locator.

## Notes & gotchas

- Scene entries require `window.__sigma`; if it isn't exposed, resolve the
  instance first (probe a global / inject an init-script before load).
- Every scene entry is `a11y_flag: true` — canvas nodes are absent from the
  accessibility tree (a genuine a11y gap, recorded, not worked around).
- Verify/act via `scripts/scene_click.py` (emits the `eval` + `mouse*` plan) or the
  generated `sigma-demo.pom.ts` async methods (`await graph.alice()`).
